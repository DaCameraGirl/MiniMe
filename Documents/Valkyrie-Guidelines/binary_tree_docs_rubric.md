# Binary Tree Documentation Rubric

## Core Criteria

1. (20 points) Response lists time/space complexity for at least one core operation (insert/find/delete)  
   Example: "Binary tree find operation has O(h) time complexity where h is tree height"

2. (20 points) Response specifies at least three edge cases requiring documentation  
   Example: "Document handling of null inputs, duplicate values, and memory limits"

3. (20 points) Response includes example code snippets for basic API usage  
   Example: "Show initialization, insertion of 3 nodes, and find operation"

4. (20 points) Response proposes documentation versioning approach  
   Example: "Add @since tags for API methods and maintain CHANGELOG.md"

5. (10 points) Response warns about at least one common misuse pattern  
   Example: "Document that insertion order affects tree balance"

6. (10 points) Response suggests documentation template sections  
   Example: "Include Overview, API Reference, Examples, and Performance sections"
